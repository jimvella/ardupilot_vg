Drone autopilot flight logging

For aerobatic and loss of control training flights

A technical ecosystem and collective experience in the form of open source software and commodity hardware has developed around drones that we can tap into for relatively low cost yet precise aircraft flight path logging. ArduPilot is an example of such software which at its heart has an "advanced sensor fusion algorithm that can extend the GPS based positioning with inertial navigation to deliver a smooth and accurate flight path and attitude log with 15-25 points per second. This compares to 1-2 per second (with no attitude information) of a typical GPS only system." [1]. CubePilot is an example of commodity hardware that can host software such as ArduPilot.

ArduPilot Plane BlackBox Logger

ArduPilot has specific firmare* designed for the BlackBox logger usecase. Note this is a build time option, but there is a community build server available for building.

* In ArduPilot nomenclature 'firmware' means software that runs on the drone - as opposed to software that runs on the ground station.

CubePilot Hardware

A suggested configuration is the Cube Orange+, Mini Carrier Board, and Here4 GPS (roughly $1000 AUD).

Analysing Logs

The logs recorded by the autopilot are called Dataflash logs and can be identified by the '.bin' file extention. The easiest way to review ArduPilot logs is the web based UAV Log Viewer application. Details of what is logged can be found in the ArduPilot documentaion.
